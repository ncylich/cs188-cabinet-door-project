Abstract

We study the OpenCabinet task in RoboCasa, where a PandaOmron mobile manipulator must open a kitchen cabinet door in previously unseen kitchen layouts using imitation learning. Our project began from the course starter code and evolved into a larger empirical study of behavioral cloning and diffusion-policy variants under severe data constraints. The main result was that representation quality mattered more than model complexity early on: low-dimensional proprioception alone could not solve the task, door-centroid oracle features helped but were insufficient, and handle-relative features were the critical signal. On the original 107-demo dataset, our best model was a BC U-Net using proprioception, handle position, and handle-to-end-effector relative displacement, achieving 44 / 100 = 44% success under a relaxed one-door-open criterion. After scaling to a combined 607-demo dataset, a diffusion U-Net surpassed BC and reached 49% success, showing that diffusion was more data-hungry but stronger once enough demonstrations were available.

Task and Setting

OpenCabinet is difficult because the robot must localize the cabinet, reach the handle, make effective contact, and pull the door open in randomized kitchens rather than a single memorized scene. Each evaluation episode uses an unseen kitchen layout, so the policy must generalize across geometry and placement rather than memorize one trajectory.

Task RoboCasa OpenCabinet Robot PandaOmron mobile manipulator
Original dataset 107 demos, 37,492 timesteps Scaled dataset 607 demos
Observations 16-D proprioception plus 3 RGB cameras Actions 12-D arm, base, rotation, gripper, and mode control

We focused on three questions: what observation features are actually necessary, which imitation-learning method works best at this scale, and whether the bottleneck is localization, approach, or grasp-and-pull execution.

Approach

The project progressed through a sequence of controlled ablations: low-dimensional BC and diffusion baselines, frozen visuomotor features, oracle door-state inputs, handle-relative features, and finally data scaling. Across these stages we implemented both behavioral cloning and diffusion-policy pipelines, compared MLP, 1D U-Net, and Transformer backbones, used DDPM and DDIM schedulers, and evaluated action chunking with short observation histories.

  1. Low-dimensional baselines. We first verified BC and diffusion pipelines on the raw 16-D proprioceptive state and confirmed that action prediction worked technically, even though task success remained at 0%.
  2. Visuomotor features. We added frozen image encoders and spatial pooling to test whether RGB alone could provide enough localization signal at this dataset scale.
  3. Oracle door-state diagnosis. We replayed episodes in simulation to extract door position and orientation, separating perception from control and producing the first nonzero success.
  4. Handle-relative representation. We replaced the door centroid with the actual MuJoCo handle site and handle-to-EEF displacement, which aligned the observation with the true grasp target.
  5. BC versus diffusion under scaling. Finally, we compared the best handle-aware BC and diffusion variants on the original 107-demo dataset and on the larger combined 607-demo dataset.

The crucial representation change was moving from door-centroid features to the actual MuJoCo handle site plus handle-to-end-effector relative displacement. That change aligned the observation with the actual grasp target and unlocked the first strong task success.

Results

The main empirical pattern was consistent. Representation dominated early performance. BC was best in the low-data regime. Diffusion only won after scaling the dataset.

Repository and Environment

The checked-in repository is still partly a course starter project. The README remains a starter-style walkthrough for installation, environment exploration, dataset download, training, and evaluation. The CURRENT_RESULTS.md file is a chronological experiment log rather than a polished final report.

Based on the current install.sh, the intended environment creates a local .venv, clones and installs robosuite and robocasa, installs PyTorch plus imageio[ffmpeg], pins numpy==2.2.5, and downloads roughly 10 GB of kitchen assets.
